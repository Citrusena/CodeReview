1. GitHub 오픈소스 (가장 추천)
방법:
GitHub에서 “good first issue” 라벨 검색
초보자도 기여 가능한 이슈들
예: https://github.com/topics/good-first-issue
Python 프로젝트 필터링
시작하기:
관심 있는 작은 프로젝트 찾기 (star 100-1000개 정도)
Issues 탭 → “good first issue” 태그
코드 읽어보고 이해하기
개선점 있으면 PR 날리기
없어도 “코드 읽는 연습” 됨

2. 백준 다른 사람 풀이 보기
방법:
	∙ 문제 맞은 후 → “다른 사람 풀이” 클릭
	∙ 똑같은 문제를 다양한 방식으로 푼 코드들 비교
	∙ “아, 이렇게도 풀 수 있구나” 배우기
팁:
	∙ 본인 풀이보다 짧은 코드 찾기
	∙ 본인 풀이보다 빠른 코드 찾기
	∙ 왜 다른지 분석하기
3.. CodeReview StackExchange
	∙ 사이트: https://codereview.stackexchange.com/
	∙ 사람들이 “내 코드 리뷰해주세요” 올림
	∙ 영어지만 코드는 언어 무관
4. 논문 코드 찾는 법 (보너스)
많은 논문들이 GitHub에 구현 코드 공개해요:
	∙ Papers with Code: https://paperswithcode.com/
	∙ 논문 제목 + “github” 구글 검색
	∙ 논문 마지막 “Code Availability” 섹션 확인
예시:
	∙ 논문: “Attention Is All You Need”
	∙ 검색: “transformer github implementation”
	∙ 결과: 수십 개 구현체 나옴

YouTube 코드 리뷰 영상
	∙ “Python code review” 검색
	∙ 다른 사람이 리뷰하는 거 보면서 배우기

백준 문제 풀기 > 2달안에 단계별로 문제풀기 1회전.
 > 월화수목 12문제만 풀기. (하루 깃 커밋 12개)
 > 금 22문제만 풀기. (금요일 깃 커밋 22개)
